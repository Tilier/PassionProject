Lesson #2: CSS Basics - Part One

From: CSS

CSS is made out of properties and selectors.

Here I’ll explain to you what selectors are.

In CSS, you have to specify what you want to style. For instance, you might want to color one div green, and one div blue. How do you do this? You use selectors. Now, in HTML, you can give any element classes and IDs. You do this simply by adding in class=”something” or id=”something” in the element. For instance, div class=”class” id=”id”. I’d like you to go to the CSS tab in the example and take a look at the CSS. Take a close look, and then I’ll explain in further detail.

To reference different elements, you need to specify which element(s) that you are styling. To reference all elements with a certain class, you use .myclass, or .example, or whatever class that you are referencing after a dot. For an ID, you want to type a # and then the ID, for example #myID or #example. And then, for any element with a certain tag, you just type the tag, like div or h1 without < or >. Then, you have to specify which properties you want that selector to style. Here’s the format you use:

.example {
    color: red;
    background: lightblue;
}

You put all the properties inside the brackets.

You can also do more specific selectors. For instance, you can smush two selectors together. Here are some examples of that:

  • .example.myclass
  • .example#myID
  • #myID.example
  • div.example
  • .example.myclass.hello
  • .example#myID.myclass

The examples above mean anything that has falls under all of those specifiers. For instance, .example.myclass would have to be any tag whose classes included example and myclass. The thing is, you can assign numerous classes to an element. For instance, you can write: <div class=”example myclass”>Hello!</div>. All you have to do is put a space between the classes.

And, lastly, we’ll discuss using parent elements for specifications.

Note: if you’re wondering what parent elements are, go here: https://passionproject.ml/lessons/html/html-basics#parentelements. Examples of using parents elements for specifications:

Examples of using parents elements for specifications:

  • .parent .child
  • #parent .child
  • #grandparent .parent #child
  • div .grandparent .child
  • ul li
  • div h1 span .child

The first example, .parent .child, means any tag with the class of child inside any tag with the class of parent. Now, .parent doesn’t have to be the direct parent. It can be the grandparent, or the great-grandparent — heck, it could be the great-great-great-great-great-great-grandparent, for all we know!
